# MoonGuard 项目申报书

## 1. 项目名称和 GitHub 仓库地址

项目名称：MoonGuard：MoonBit 公共 API 兼容性与语义版本检查工具

GitHub 仓库地址：https://github.com/918154429/moonguard

## 2. 项目简介

MoonGuard 是一个面向 MoonBit 包生态的公共 API 兼容性检查工具。它读取
MoonBit 工具链生成的 `.mbti` / `pkg.generated.mbti` 接口文件，解析公共
声明，对比两个版本之间的 API 变化，并给出 `major`、`minor` 或 `patch`
级别的 SemVer 建议。项目目标是帮助 MoonBit 包作者在发布前发现破坏性变更，
降低下游用户升级风险。

## 3. 项目方向与适用场景

项目方向：MoonBit 工程基础设施 / 包生态质量工具 / CI 发布检查工具。

适用场景包括：MoonBit 库发布前检查公共接口变化；Pull Request 中自动识别
删除 API、修改签名等破坏性变更；发布流程中验证计划版本号是否满足 SemVer
要求；生成 Markdown 报告用于 PR comment 或 release note；输出 JSON 报告供
CI 和后续工具链消费。

MoonBit 生态仍处于早期，但已具备 mooncakes 包注册中心、SemVer 版本管理和
公开接口文件。GitHub 上也能检索到大量 `pkg.generated.mbti` 文件。MoonGuard
正是围绕这些已有生态产物，补齐 API 兼容性检查这一层基础设施。

## 4. 已实现的核心功能

- 解析 `.mbti` / `pkg.generated.mbti` 中的公共声明。
- 支持 `fn`、`type`、`typealias`、`struct`、`enum`、`trait`、`impl`、`let`、
  `const`，以及 struct field、enum constructor、trait method、`suberror` 等
  常见接口形态。
- 构建规范化 API 模型，忽略空白、注释、声明顺序等非语义差异。
- 对比两个接口快照，识别新增、删除和签名变化。
- 根据变化给出 `major` / `minor` / `patch` 建议，并检查计划版本号是否满足要求。
- 支持单文件比较、目录级 package 比较、Markdown/JSON 报告、ignore rules、
  配置文件和 GitHub Actions CI。

## 5. 当前验证结果

- `moon check`、格式检查、162 个默认目标测试和 163 个 JS 目标测试全部通过。
- 覆盖率为 1900/2183 行（87.0%）。
- 已引入 15 个公开仓库的真实 `.mbti` 快照，固定到完整 commit SHA，并记录
  来源、许可证和 SHA-256。
- 15 个真实样本共建模 6819 个 API 条目，全部实现零 `unknown` 声明和零快照诊断。
- GitHub Actions 覆盖报告、目录扫描、版本校验、release plan 和退出码检查。

## 6. 原创性说明

本项目为原创 MoonBit 工程基础设施项目，不是移植项目。项目参考了成熟生态中
API 兼容性检查工具的设计思路，例如 Rust `cargo-semver-checks`
（https://github.com/obi1kenobi/cargo-semver-checks）、Go `apidiff`
（https://pkg.go.dev/golang.org/x/exp/apidiff）和 TypeScript API Extractor
（https://api-extractor.com/），但不复用、不复制其源代码。

MoonGuard 的输入格式、API 模型、兼容性规则和 CLI 工作流均围绕 MoonBit
`.mbti` 接口文件、MoonBit package 和 mooncakes 发布流程重新设计。项目实现
语言为 MoonBit，许可证为 Apache-2.0。
